# Contribuindo

Obrigado por colaborar com a plataforma Always Free!

## Como começar

1. Faça um fork ou crie uma branch a partir de `main`.
2. Rode `npm install` e `npm run dev` para validar localmente.
3. Execute `npm run lint && npm test` antes de abrir PR.

## Estilo de código

- Use TypeScript estrito.
- Execute `npm run format` (Prettier) e mantenha imports organizados.
- Comentários apenas quando realmente ajudam a explicar decisões.

## Commits

- Preferência por Conventional Commits (`feat:`, `fix:`, `docs:` etc.).
- Descreva impacto usuário + testes executados no corpo da PR.

## Issues

- Use o template de issues (ver `ISSUES_TO_AUTOGENERATE.md`).
- Priorize melhorias de monetização/automação e redução de custos.

## Segurança

- Nunca faça commit de chaves reais.
- Se notar comportamento suspeito (fraude/abusos), abra issue privada ou contate o maintainer.
